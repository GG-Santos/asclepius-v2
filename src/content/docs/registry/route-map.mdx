Use this map when tracking whether a workflow has been verified end to end.
Registry routes are stable enough for daily operations. Courses, Publishing, and
Administration routes are listed so admins can find them, but those areas remain
experimental or under development.

## Registry Routes

| Area | Route | Use |
| --- | --- | --- |
| Overview | `/dashboard` | Registry health, renewal risk, recent verification lookups, and quick actions. |
| Batches list | `/dashboard/batches` | Search, filter, create, view, edit, and delete empty batches. |
| Batch detail | `/dashboard/batches/[id]` | Review roster, graduates, batch metrics, and public cohort context. |
| Batch edit | `/dashboard/batches/[id]/edit` | Edit cohort metadata, professor, description, media, grading scheme, quiz definitions, and proficiency rows. |
| Students list | `/dashboard/students` | Search active students and failed students, then view, edit, graduate, restore, or delete records. |
| New student | `/dashboard/students/new` | Add a training record. |
| Student detail | `/dashboard/students/[id]` | Inspect one training record before editing or graduating. |
| Student edit | `/dashboard/students/[id]/edit` | Edit training identity, batch assignment, photo, quizzes, and practical scores. |
| Graduates list | `/dashboard/graduates` | Search credential records, renew, view, edit, archive, restore, or delete. |
| New graduate | `/dashboard/graduates/new` | Add verified historical or manual graduate records. |
| Graduate detail | `/dashboard/graduates/[id]` | Inspect one credential, artifacts, public link, renewal action, archive action, delete action, and portal account manager. |
| Graduate edit | `/dashboard/graduates/[id]/edit` | Edit public credential fields and proficiency records. |
| Recertified export | `/dashboard/graduates/recertified` | Filter renewed records and download front/back ID card ZIP exports. |
| Public verification | `/verify/[lcn]` | Public credential result for one LCN. |
| Public cohort | `/cohorts/[id]` | Public cohort page linked from registry media and graduate context. |

## Tracking Rule

For every Registry change, track three points:

1. The admin route where the action happened.
2. The record route that shows the saved result.
3. The public or export route affected by the change.

## Troubleshooting

| Symptom | Check |
| --- | --- |
| A link opens the wrong area | Use the left sidebar label first, then compare the URL against this map. |
| A record is missing from a list | Confirm the status tab or server filter, such as Students vs Failed or Graduates vs Archived. |
| A public page does not match the admin record | Reopen the graduate detail route, confirm the saved fields, then refresh `/verify/[lcn]`. |
