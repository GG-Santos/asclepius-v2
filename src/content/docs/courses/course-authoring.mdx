not Registry credential changes.

<OpenRoute href="/dashboard/courses/new">Open New Course</OpenRoute>

## Create a Course Shell

1. Open Courses.
2. Choose **New course**.
3. Enter Title, optional Slug, Estimated time, cover image, and Summary.
4. Save the course shell.
5. Add modules and items from the course editor.

## Editor Areas

| Area | Use |
| --- | --- |
| Course details | Title, slug, estimated time, cover, summary, and certificate setting. |
| Curriculum | Modules and ordered items graduates work through. |
| Publish controls | Draft, published, and archived course state. |
| Preview | Graduate-facing test view. |
| Roster | Graduate enrollment and progress review. |

## Publish Checklist

- The course has at least one useful module.
- Published modules contain published items.
- Pages, links, files, videos, and quizzes open in preview.
- Sequential locks and prerequisites are intentional.
- Completion certificate behavior has been tested.
- Registry credential state was verified separately when portal access matters.

## Troubleshooting

| Symptom | Cause | Fix |
| --- | --- | --- |
| Preview is empty | Published course has no published items. | Publish the module and at least one item. |
| Quiz edits do not affect a completed attempt | Existing progress may already be recorded. | Test with a fresh graduate or reset progress through the appropriate admin path. |
| Completion certificate is missing | Certificate is disabled or requirements are incomplete. | Check the course setting and completion path. |

<DocNote title="Publishing is visible to graduates" tone="warn">
Do not publish draft learning content just to test navigation. Use preview until
the course is ready for graduate access.
</DocNote>
